1.)Preplanning
- การเตรียมการ เราจะต้องนึกถึง requirement ทุกอย่างที่ได้รับมา และปัญหาที่อาจเกิดขึ้นได้หากมีมีการ เพิ่มข้อมูลหรือเพิ่ม Column ควรที่จะ design ให้เกิดปัญหาเมื่อมีการแก้ไข Table ให้น้อยที่สุด
2.)Understanding the purpose of data
- การทำความเข้าใจ DataBase ผู้ออกแบบต้องเข้าใจ requirement เพื่อออกแบบให้สอดคล้องกับ requirement อย่างเหมาะสมที่สุด โดยต้องคำนึงถึง ลักษณะของข้อมูล วิธีการได้มา ความถี่ในการจัดเก็บและดึงข้อมูล ปริมาณของข้อมูล 
3.)Normalization
- มีหลักการสำคัญบางประการในการออกแบบที่มีความสำคัญในการทำให้ฐานข้อมูลทำงานได้อย่างเหมาะสมที่สุด หนึ่งในหลักการเหล่านี้คือการทำให้เป็นมาตรฐาน หากเราทำให้เป็นอันเดียวกันกับมาตราฐานจะทำให้เพื่อนในทีมสามารถทำงานได้ง่ายเพราะสามารอ่านแล้วทำความเข้าใจได้เลย
4.)Prevent redundant records
- การออกแบบควรที่จะป้องกันการซ้ำซ้อนของข้อมูล เนื่องจากว่าการเพิ่มขนาดของฐานข้อมูลโดยไม่จำเป็น จะลดประสิทธิภาพและเพิ่มความเสี่ยงที่ข้อมูลจะเสียหาย
5.)Indexing
- เนื่องจากว่าเมื่อจำนวนระเบียนแถวในตารางเพิ่มขึ้น เวลาที่ใช้ในการสืบค้นข้อมูลเหล่านี้จะเพิ่มขึ้นเรื่อยๆ การทำ Indexing เป็นการทำให้สามารถดึงข้อมูลได้ไวขึ้น 
6.)Domain values and table management
7.)Consistent naming conventions
- หลักการตั้งชื่อให้สอดคล้องกัน ส่วนใหญ่ใช้กับ ForeignKey เพื่อให้คนในทีมสามารถเข้าใจได้ว่าชื่อตัวแปรนี้มาจากไหนแล้วใช้งานได้อย่างถูกต้อง
8.)Documentation
- การเขียน Documentation คือการเขียน หน้าตาของข้อมูลที่รับส่งระหว่าง Back end และ front end เพื่อเป็นตัวกลางในการสื่อสารและทำความเข้าใจให้เข้าใจตรงกันเพื่อให้ไม่เกิดการแก้ไขในภายหลังมากมายนัก
9.)Usage testing
- เมื่อทำการทดสอบอย่างละเอียดและครอบคลุมก่อนที่ฐานข้อมูลจะเผยแพร่ จะช่วยลดจำนวนและขนาดของความผิดพลาดได้อย่างมากหลังจากการปรับใช้กับการใช้งานจริง

ปัญหาที่เคยพบ
- มีการเพิ่ม Column ของ DataBase เพิ่มเติมในภายหลัง
- หน้าตาของ DataBase relational ยังไม่พร้อมให้กับ Front end ใช้งาน
- มีการเพิ่ม table ขึ้นมาใหม่ที่ Relational กับ Table 

วิธีการแก้ปัญหา 
- ใช้ MySQL Workbench ในการเพิ่ม Column เพื่อไม่ให้ข้อมูลเดิมที่มีอยู่เสียหาย ทำการ Update Model และ ER Diagram
- เอาข้อมูลที่ Query ได้มาทำการ Map เพื่อจัดการหน้าตาของ DataBase ใหม่ให้ใช้งานง่าย
- ใช้ MySQL Workbench ในการจัดการแก้ไขข้อมูล relational ใน DataBase ทำการ Update Model และ ER Diagram
